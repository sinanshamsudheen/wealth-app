Navigation Strategy — General Nav + Module Nav

How to handle the global sidebar alongside per-module navigation

The Problem

Two competing navigation needs
Global Navigation (always)
  • Home / Modules
  • Your Daily
  • Chat
  • Agents
  • Administration
Module Navigation (contextual)
  • Deals: Pipeline, Opportunities, Allocations, Reports
  • Engage: Clients, Prospects, Interactions, Pipeline
  • Admin: Company Profile, Branding, Roles & Access
  • Insights: Dashboards, Reports, Alerts, Data Sources
Problem: Two sidebars = confusing. One sidebar trying to hold everything = cluttered. How do we solve this?

Pick an approach

A

Contextual Sidebar — Sidebar morphs when you enter a module

The sidebar changes its contents based on where you are. At the top level, it shows global nav. When you click into Deals, it transforms into Deals-specific nav with a "back" arrow to return to global nav.

Pros

  • Clean — one sidebar, never two
  • Full width for module nav items
  • Used by Notion, Linear, Slack
  • Scales well as modules grow

Cons

  • Switching between modules requires going "back" first
  • Loses sight of global nav while in a module
B

Icon Rail + Module Panel — Thin global rail + expandable module nav

A permanent thin icon rail (56px) on the far left shows global items (Home, Chat, Agents, each module icon). Clicking a module icon opens a second panel (220px) with that module's sub-navigation. The panel collapses when not needed.

Pros

  • Global nav always visible as icons
  • Quick module switching via icon rail
  • Module nav has dedicated space
  • Used by VS Code, Discord, Teams

Cons

  • Takes more horizontal space when expanded
  • Two visual layers to learn
C

Top Bar Module Switcher + Sidebar — Module switching in header, sidebar is always module-specific

The top navigation bar has a module switcher dropdown (like the current ModuleSwitcher). When you select a module, the sidebar shows that module's navigation. Global items (Chat, Agents, Your Daily) live in the top bar or as persistent icons at the bottom of the sidebar.

Pros

  • Clean single sidebar
  • Module context always clear from top bar
  • Global items accessible without leaving module
  • Used by Salesforce, HubSpot, Jira

Cons

  • Module switching requires top bar interaction
  • Need to design persistent global items carefully

My Recommendation

Why I'd suggest Option A (Contextual Sidebar)

Option A is the cleanest for your use case because:

  • Users spend most time inside a single module — they don't constantly switch
  • The product requirement doc confirms "only two pages needed" (workspace + dashboard) — users want focus, not multi-module juggling
  • Keeps maximum content width — no icon rail eating space
  • Global items (Chat, Agents, Your Daily) can pin at the sidebar bottom, always accessible
  • Module switcher in top bar provides quick switching when needed (already built)

Essentially: Option A sidebar + Option C's top bar module switcher gives you the best of both worlds.
